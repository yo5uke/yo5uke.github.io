CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Project Overview

This is a personal Japanese-language website built with Quarto and R, focusing on R programming tips, GIS visualization, software projects, and blog posts. The site is hosted at https://yo5uke.com/ and deployed via GitHub Pages.

Tech Stack

  • R: 4.5.2 (managed via renv)
  • Quarto: 1.9.9 (for website generation)
  • Python: 3.x (managed via venv in .venv/)
  • DVC: Data version control (via Python, see requirements.txt)
  • renv: R package management (see renv.lock)

Essential Commands

Building and Previewing

# Preview the website locally (with live reload)
quarto preview

# Render the entire site
quarto render

# Render a specific page
quarto render pages/tips/241117_ggplot2/index.qmd

R Environment

# Restore R package environment
R -e "renv::restore()"

# Update renv.lock after installing new packages
R -e "renv::snapshot()"

Python Environment

# Activate Python virtual environment (Windows)
.venv/Scripts/activate

# Activate Python virtual environment (Unix/macOS)
source .venv/bin/activate

# Install Python dependencies
pip install -r requirements.txt

Data Version Control (DVC)

# Pull data files from remote storage
dvc pull

# Push updated data to remote storage
dvc push

# Check status of DVC-tracked files
dvc status

Note: DVC is installed via Python (requirements.txt), not globally. The data/ directory (1.5GB, 233 files) is tracked by DVC and excluded from git via .gitignore.

Repository Structure

├── pages/              # Content organized by type
│   ├── tips/          # R programming tips (main content)
│   ├── blog/          # Blog posts
│   ├── software/      # Software/app showcases
│   └── gis_in_r/      # GIS visualization tutorials
├── assets/            # Static assets (icons, images)
├── components/        # Reusable UI components (HTML partials)
├── styles/            # Custom SCSS and CSL citation styles
├── data/              # Data files (DVC-tracked, ~1.5GB)
├── _freeze/           # Quarto computation cache
├── _site/             # Generated website output
├── _quarto.yml        # Main Quarto configuration
├── _brand.yml         # Brand/theme customization (colors, fonts, CSS)
├── _language.yml      # Japanese language customizations
└── renv.lock          # R package dependencies

Content Organization

  • Each page/post is in its own directory under pages/{type}/{date}_{slug}/index.qmd
  • Featured content carousel controlled by pages/tips/carousel.yml
  • Tips listing page: pages/tips/index.qmd (auto-generates from all **/index.qmd in tips/)
  • Site metadata and per-page configuration in YAML frontmatter

Key Configuration Files

_quarto.yml

  • Website structure, navbar, footer
  • Google Analytics integration
  • Freeze execution mode (freeze: auto - caches computation results)
  • Output directory: _site/

_brand.yml

  • Color palette (mint, coral-pink, gray themes)
  • Typography (Google Fonts: Zen Maru Gothic, Zen Kaku Gothic New, Fira Code)
  • Custom Bootstrap rules and component styling
  • Navbar, title banner, carousel, callout, and code block styling

_language.yml

  • Japanese translations for Quarto UI elements (footnotes, search, callouts)

Development Workflow

Adding New Tips/Blog Posts

  1. Create a new directory: pages/tips/YYMMDD_{slug}/
  2. Add index.qmd with YAML frontmatter (title, date, categories, description, image)
  3. Add images to same directory or image/ subdirectory
  4. Quarto will automatically include it in the listing pages
  5. To feature on homepage carousel, add entry to pages/tips/carousel.yml

Working with R Code

  • R environment is managed by renv (see .Rprofile which sources renv/activate.R)
  • Code execution is cached in _freeze/ directory (controlled by freeze: auto)
  • To force re-execution of a specific document, delete its freeze cache: rm -rf _freeze/pages/tips/{slug}/

Working with Data

  • Large data files are in data/ and tracked by DVC (not git)
  • DVC configuration in .dvc/ and data.dvc
  • Run dvc pull before working with data-dependent pages
  • After updating data: dvc add data/ then dvc push

Styling and Theming

Custom CSS Classes (defined in _brand.yml)

  • .libre-baskerville-regular - Serif font for headings
  • .fira-code - Monospace font for inline code
  • Heading underlines: h1 (mint bar), h2 (mint fade), h3 (light gray)
  • Carousels scale images to 70% width
  • Social share buttons (Bluesky, X/Twitter) with hover effects

Component Templates

  • components/ui/button.html - Reusable button component
  • components/ui/twitter.html - Twitter embed component
  • components/ui/margin-header.html - Margin header layout

Important Notes

  • Language: All content is in Japanese; UI labels customized in _language.yml
  • License: Content under CC BY-SA 4.0
  • Deployment: Automated via GitHub Pages (pushes to main branch trigger rebuild)
  • Freeze mode: Computation results are cached; delete _freeze/{path} to force re-execution
  • Python dependencies: Include DVC, Jupyter, Selenium, pandas, polars, matplotlib
  • Secrets: .secrets/ and .Renviron are gitignored (credentials for DVC remote, APIs, etc.)

Common Tasks

Update ggplot2 guide or other frequently updated pages

# Delete the freeze cache to force re-execution
rm -rf _freeze/pages/tips/241117_ggplot2/
quarto render pages/tips/241117_ggplot2/index.qmd

Add a new R package dependency

# In R console
install.packages("packagename")
renv::snapshot()  # Update renv.lock

Preview changes before committing

quarto preview  # Opens browser with live reload
# Make changes, browser auto-refreshes
# Ctrl+C to stop

Check which files will be rebuilt

quarto render --debug
# Or delete specific freeze caches to force rebuild
